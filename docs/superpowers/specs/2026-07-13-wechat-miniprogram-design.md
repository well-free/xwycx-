# xwycx.xyz 微信原生小程序设计

## 1. 目标与范围

为一次性用品订单交易系统增加微信原生小程序端，提供完整采购闭环和管理员运营能力。小程序复用现有 Spring Boot API、用户、商品、采购订单、支付、退款、库存和消息能力，不建立平行的业务体系。

首版包含：

- 微信授权登录和手机号验证码登录，两种身份绑定为同一用户。
- 首页、分类、商品详情、购物车、确认订单、地址管理、支付、订单列表、订单详情、退款申请和个人中心。
- 管理员经营概览、商品管理、库存调整、订单发货、退款审核和支付记录。
- 开发环境模拟微信支付，生产环境切换微信支付 API v3。
- 微信开发者工具可直接导入的原生小程序工程。

首版不包含直播、拼团、优惠券、积分、分销、多商户结算、即时聊天和复杂营销规则。

## 2. 总体架构

小程序工程放在仓库根目录 `wechat-miniprogram/`，与 Spring Boot 工程独立。小程序只通过 HTTPS REST API 与后端通信，统一使用 `X-Session-Token` 传递当前项目登录态。

后端继续作为业务事实源：

- Spring Boot 负责身份、权限、商品、购物车、订单、库存、支付和退款编排。
- MySQL 保存用户、微信身份、地址、购物车、订单、支付和审计数据。
- Redis 保存会话、验证码、限流计数、热点缓存和必要的购物车缓存。
- RocketMQ 处理订单超时关闭和异步业务事件。
- Nginx 在生产环境为 `xwycx.xyz` 提供 HTTPS 和反向代理；本地开发不要求安装 Nginx。

小程序代码按可迁移架构组织：

```text
wechat-miniprogram/
  app.*
  pages/
  components/
  services/
  store/
  types/
  platform/
  utils/
  assets/
```

页面只处理展示和交互。REST 调用集中在 `services/`，登录态和购物车状态集中在 `store/`，微信特有的登录与支付 API 集中在 `platform/`。后期迁移 Taro 时，核心接口类型和纯业务逻辑可以复用，主要重写 WXML/WXSS 和页面生命周期。

## 3. 用户与双登录

### 3.1 微信授权登录

小程序调用 `wx.login` 获取一次性 `code`，后端通过微信 `code2Session` 换取 `openid` 和 `session_key`。后端根据 `appid + openid` 查找微信身份并创建或恢复项目会话，绝不把 `session_key` 返回前端。

新增 `wechat_identities` 表，核心字段为：

- `id`
- `user_id`
- `appid`
- `openid`
- `unionid`
- `created_at`
- `updated_at`

`appid + openid` 必须唯一。

### 3.2 手机验证码登录与绑定

现有短信验证码登录继续保留。微信用户首次需要手机号能力时，可通过短信验证码绑定已有手机号用户。若手机号已经属于另一个用户，必须再次验证手机号后执行受控合并，不允许静默覆盖用户关系。

合并后的商品订单、地址、购物车和支付数据都归属于同一个 `users.id`。管理员角色继续保存在用户表，微信身份不能自行提升角色。

### 3.3 新接口

```http
POST /api/auth/wechat/login
POST /api/auth/wechat/bind-phone
```

`POST /api/auth/wechat/login` 接收小程序 `code`。本地 profile 使用可配置的模拟身份；生产 profile 必须配置小程序 AppID 和 AppSecret，并调用微信服务端接口。

## 4. 页面与导航

用户端底部导航固定为四项：

- 首页：推荐商品、分类入口和运营内容。
- 分类：分类筛选、搜索和商品列表。
- 购物车：数量调整、删除、选中和结算。
- 我的：登录状态、订单入口、地址管理和退出登录。

二级页面包括商品详情、确认订单、支付结果、订单列表、订单详情、退款申请、地址列表和地址编辑。

管理员登录后在“我的”页面显示管理员工作台入口。管理员工作台包含经营概览、商品管理、库存管理、订单处理、退款审核和支付记录。普通用户不渲染入口，所有 `/api/admin/**` 接口仍由后端验证 `ADMIN` 角色，前端隐藏不视为安全措施。

## 5. 购物车与地址

新增 `cart_items` 表，唯一约束为 `user_id + product_id`，保存商品数量和更新时间。购物车商品价格只用于展示，下单时必须从数据库重新读取商品状态、当前价格和库存。

复用并补全 `shipping_addresses`，至少包含收货人、手机号、省市区、详细地址、默认地址标记和审计时间。订单创建时把完整收货信息保存为订单快照，后续修改地址簿不能改变历史订单。

新增接口：

```http
GET    /api/cart
POST   /api/cart/items
PUT    /api/cart/items/{productId}
DELETE /api/cart/items/{productId}
GET    /api/addresses
POST   /api/addresses
PUT    /api/addresses/{id}
DELETE /api/addresses/{id}
```

购物车和地址接口全部要求登录，并校验数据归属。

## 6. 下单与库存

小程序复用 `customer_orders` 和 `customer_order_items`。订单创建步骤为：

1. 校验用户、地址、商品上架状态和数量。
2. 从数据库读取商品当前价格，生成订单项价格快照。
3. 使用现有 Redisson 前置锁和 MySQL 乐观锁扣减库存。
4. 写入采购订单、订单项和库存日志。
5. 发布订单超时关闭消息。
6. 从购物车结算时，仅删除成功下单的选中商品。

任何步骤失败时事务回滚，不产生半成品订单。未支付订单取消或超时关闭后必须幂等释放库存。

## 7. 两阶段微信支付

### 7.1 网关结构

在现有 `PaymentGateway` 下增加微信小程序支付实现：

- `MockWechatPaymentGateway`：仅 local profile 或明确的 sandbox 模式启用。
- `WechatPayV3Gateway`：仅生产模式启用，调用微信支付 JSAPI 下单、查询和退款接口。

统一支付响应为小程序提供 `timeStamp`、`nonceStr`、`package`、`signType` 和 `paySign`。模拟模式返回可识别的模拟参数，并提供受环境保护的模拟成功入口。

### 7.2 状态流转

```text
PENDING_PAYMENT
  -> 创建预支付单
  -> 小程序 wx.requestPayment
  -> 后端接收统一支付回调
  -> 验签、金额校验、订单归属校验、幂等校验
  -> payment_orders = SUCCESS
  -> customer_orders = PAID
```

前端支付成功回调只代表微信客户端完成交互，不能直接把订单改为已支付。订单最终状态必须以后端回调或主动查询微信支付结果为准。网络中断后，小程序进入订单详情轮询最终状态。

模拟支付也调用统一的后端支付回调处理逻辑，保证开发和生产共享相同状态机。

### 7.3 生产配置

生产环境通过环境变量配置：

- 小程序 AppID 和 AppSecret。
- 微信支付商户号。
- 商户证书序列号和商户私钥路径。
- API v3 密钥。
- 支付通知与退款通知 URL。

私钥和证书不得提交到 Git。`https://xwycx.xyz` 由 Nginx 提供有效 HTTPS 证书，并在微信公众平台配置为合法 request 域名。网站 HTTPS 证书与微信支付商户证书用途不同。

## 8. 权限、限流与异常处理

- 所有用户订单、地址、购物车、支付和退款接口验证会话及数据归属。
- 所有管理接口验证 `ADMIN` 角色。
- 微信登录、短信发送、验证码登录、下单和支付创建接入 Redis/Lua 限流。
- 微信登录失败、支付预下单失败不改变订单状态。
- 重复支付回调通过渠道交易号和回调幂等键只处理一次。
- 签名、金额或商户号不一致时拒绝回调并记录审计日志。
- 商品下架、库存不足和地址无效返回稳定的业务错误码，供小程序展示明确提示。
- 支付结果未知时不显示支付失败，而是进入“确认中”并查询订单最终状态。

## 9. 测试与验收

### 9.1 后端测试

- 微信 code 登录创建身份并恢复会话。
- 微信身份绑定手机号及冲突保护。
- 普通用户不能访问管理员接口。
- 购物车增删改、归属隔离和下单后清理。
- 地址归属、默认地址切换和订单地址快照。
- 下架商品与库存不足不能下单。
- 取消或超时订单只释放一次库存。
- 模拟微信支付成功走统一回调路径。
- 微信支付重复通知只处理一次。
- 微信回调签名、金额和商户号异常被拒绝。

### 9.2 小程序验收

- 微信开发者工具可直接导入并启动。
- 微信模拟登录和手机号验证码登录都可用。
- 登录态过期后自动回到登录页，原目标页可恢复。
- 用户能完成浏览、加购物车、填写地址、下单、模拟支付、查询订单和申请退款。
- 管理员能管理商品、调整库存、发货和审核退款。
- 普通用户看不到管理员入口，直接请求管理接口返回 403。
- 页面在常见手机宽度无横向溢出、遮挡和不可点击控件。

### 9.3 生产验收

- `https://xwycx.xyz` 证书有效，HTTP 自动跳转 HTTPS。
- 域名已备案并配置为微信小程序合法服务器域名。
- 生产环境禁用模拟登录和模拟支付。
- 微信支付 JSAPI 下单、支付通知、订单查询和退款通知可用。
- 重启应用后用户、购物车、订单和支付数据不丢失。

## 10. 实施边界

实施按后端身份与数据能力、小程序用户端、管理员端、微信支付生产适配、联调与上线文档分阶段完成。每一阶段先增加失败测试，再实现最小通过代码，最后运行全量后端测试和小程序静态检查。
